Machine Learning / Probabilistic Prediction

Treeffuser: Probabilistic Predictions via Conditional Diffusions with Gradient-Boosted Trees

Nicolas Beltran-Velez*, Alessandro Antonio Grande*, Achille Nazaret*, Alp Kucukelbir, David Blei

NeurIPS 2024 · arXiv:2406.07658v2 · Columbia University, MSKCC, Fero Labs

Idea

Diffusion + GBT 결합

Conditional diffusion model의 score function을 gradient-boosted tree로 추정. Tabular data에 특화된 새로운 probabilistic regression 프레임워크.

Property

Non-parametric · CPU 친화

Gaussian, Poisson 등 분포 가정 없음. Multimodal/skewed/multivariate response 모두 학습. NN 없이 LightGBM만으로 학습 — laptop CPU에서 53초 안에 11.2만 샘플 학습.

Result

SOTA 능가

10개 UCI 벤치마크에서 NGBoost · iBUG · Quantile Regression · Deep Ensemble을 CRPS 기준 일관되게 능가. 하이퍼파라미터 튜닝 없이도 최고 성능.

한 줄 요약

"분포 가정이 필요 없는 회귀를 원하는데, 딥러닝은 무겁고 트리는 점추정만 한다" — Treeffuser는 conditional diffusion의 score를 LightGBM으로 학습하여, tabular data에 대한 well-calibrated probabilistic prediction을 빠르고 가볍게 제공한다.

왜 중요한가: 제조·공급망·금융·헬스케어 등 산업 현장에서 마주하는 tabular data는 multimodal·zero-inflated·long-tailed 분포가 흔하지만, 기존 probabilistic regression은 Gaussian이나 Poisson 가정에 갇혀 있었다. Treeffuser는 가정 자체를 빼고 데이터에서 분포를 직접 학습하면서도, GPU·neural network 없이 GBT만으로 동작한다는 점에서 실용성이 높다.

Background

1
STEP 1 · 문제 맥락

Probabilistic Prediction은 점추정과 무엇이 다른가

일반적인 회귀는 입력 x에 대해 단일 값 ŷ = E[y|x]를 예측한다. 반면 probabilistic prediction은 p(y|x) 자체를 학습한다 — 즉 conditional distribution 전체를 출력한다.

왜 분포가 필요한가? 불확실성 정량화, 리스크 계산, outlier 탐지 때문이다. 예를 들어 제조 공정 최적화에서는 "예측치 100"이 아니라 "95% 신뢰구간 [80, 130]"이 의사결정의 근거가 된다. 인벤토리 관리(newsvendor 문제)에서는 demand의 quantile이 최적 발주량을 결정한다.

분포를 어떻게 학습할 것인가가 다음 질문이다.

2
STEP 2 · 기존 방법의 한계

Parametric 가정의 벽

Parametric methods

NGBoost, PBGM, Bayesian NN, MC Dropout, Deep Ensembles — 대부분 Gaussian likelihood를 가정한다. 응답이 multimodal이거나 zero-inflated이거나 heavy-tailed면 모델이 잘못된 분포를 강제로 끼워 맞추게 되어 poorly calibrated한 불확실성을 출력한다.

Non-parametric methods

Quantile Regression Forests, DRF, iBUG — 가정은 적지만 일부 단점이 있다. quantile crossing 문제, similarity weight 기반 추정의 한계, 또는 multivariate response 비대응 등. 실험에서는 dataset에 따라 성능이 들쭉날쭉했다.

"가정을 빼면서도 빠르고 안정적인" 방법이 필요하다.

3
STEP 3 · 활용할 두 도구

Diffusion과 Gradient-Boosted Trees의 상보성

Diffusion Models · 분포 표현력

점진적으로 noise를 더해 단순분포로 만들고, 역으로 denoising하면서 복잡한 분포를 학습. 이미지·분자·time series 등에서 SOTA 분포 모델링. 그러나 tabular data에는 적용이 적었고, 학습이 무거웠다.

Gradient-Boosted Trees · Tabular의 강자

XGBoost, LightGBM은 categorical, missing data, mixed scale 등 tabular의 모든 까다로움을 처리하며 빠르고 robust하다. 그러나 점추정만 하고 probabilistic prediction을 직접 제공하지 못한다.

한쪽의 강점이 다른 쪽의 약점을 정확히 메운다.

4
STEP 4 · 이 논문의 제안

Treeffuser — Score를 트리로

핵심 아이디어: Conditional diffusion model을 정의하되, score function ∇y log px,t(y)를 neural network 대신 gradient-boosted tree로 추정한다. 이렇게 하면 (1) diffusion의 분포 표현력을 그대로 가져오면서 (2) tree의 tabular 친화성·CPU 학습 속도·robust성을 그대로 누린다.

결과적으로 multivariate, multimodal, skewed response를 분포 가정 없이 학습할 수 있고, M5 Walmart 데이터(11.2만 샘플)를 노트북에서 53초 안에 학습한다. 10개 UCI 벤치마크에서 CRPS 기준으로 일관되게 SOTA를 능가한다.

Method

Conditional diffusion + tree-based score estimation

3.1 Conditional Diffusion 직관

알아두면 좋을 것: Diffusion model이란?

Diffusion model은 두 개의 process로 구성된다.

(1) Forward process — 데이터 y(0) ~ π에 시간에 따라 noise를 더해간다. dy(t) = f(y,t)dt + g(t)dw. 충분히 시간이 지나면 y(T) ~ N(0, σ²I)같은 단순 분포(psimple)가 된다.

(2) Reverse process — y(T) ~ psimple에서 시작해 noise를 제거하며 데이터 분포로 되돌아간다. 이때 필요한 것이 score function ∇y log pt(y)이다. score를 알면 SDE를 풀어 sample을 얻을 수 있다.

핵심: π에서 직접 샘플링하지 못해도, score만 알면 simple distribution에서 sample → denoising으로 π에서 샘플 가능하다. 그래서 학습의 본질은 score 추정이다.

조건부로 확장하기

각 입력 x마다 별도의 forward SDE를 정의하되, drift/diffusion은 공유한다. 그러면 reverse SDE의 score는 x에 의존한다:

Conditional reverse SDE dỹx(t) = [f(ỹx, t) − g(t)² ∇ỹ log px,t(ỹx)] dt + g(t) dw̃

Score ∇y log px,t(y)를 모든 (x, y, t)에 대해 동시에 학습하는 것이 conditional score estimation 문제이다.

3.2 Score를 트리로 — Treeffuser의 정의

Reparametrization과 분리

score function S(y, t, x) ∈ ℝdy는 vector이지만, GBT는 scalar output에 강하다. 따라서 두 가지 변형을 한다:

  1. Noise-scaling reparametrization: S = U / σ(t)로 두면 U가 학습 안정성 측면에서 훨씬 잘 작동 (Appendix G ablation). 이게 핵심.
  2. 차원별 분리: 각 출력 차원 k에 대해 독립적인 GBT Uk를 학습. multivariate response는 dy개의 GBT로.
Treeffuser objective (Eq. 12) Uk* = argminUk 𝔼x,y,t,ζ [ (ζk + Uk(h(ζ,t,y), t, x))² ]

여기서 h(ζ, t, y) = m(t;y) + ζσ(t)는 noise level t에서 본 perturbed sample이고, ζ ~ N(0, I)는 raw Gaussian noise이다. 즉 GBT는 "noisy y에서 원래 noise를 복원"하도록 학습된다 — 표준 denoising score matching과 같은 형태이지만, regression target이 단순한 noise이므로 squared loss + LightGBM으로 직접 풀 수 있다.

3.3 Training & Sampling 알고리즘

1
Augment dataset
각 (x, y)에 대해 R번 t ~ U[0,T], ζ ~ N(0,I)를 sample하여 R·n개의 학습 datapoint ((h(ζ,t,y), t, x), −ζk)를 만든다.
2
Train GBTs
각 출력 차원 k마다 LightGBM을 squared loss로 학습. Treeffuser 구현에서는 LightGBM을 default로 사용한다.
3
Sample y(T)
새 x가 들어오면 y(T) ~ psimple에서 시작. VESDE 기본값 사용.
4
Reverse SDE
Euler-Maruyama solver로 ~50 step 적분. 각 step에서 학습된 U로 score 평가하여 y(0) ~ p(y|x) 1개 sample 확보. 100 sample이면 분포 추정.
왜 noise-scaling reparametrization이 핵심인가?

Diffusion에서 σ(t)는 작은 t에서 매우 작고, 큰 t에서 매우 크다. 만약 score S를 직접 학습하면 target의 scale이 1/σ(t)에 비례해 폭발/소멸하므로 GBT의 squared loss 학습이 불안정해진다.

S = U/σ(t)로 reparametrize하면 U의 target은 noise ζ 자체가 되어 모든 t에서 O(1) scale로 유지된다. Appendix G의 ablation에서 이 reparametrization 없이는 정확도가 크게 떨어진다고 보고된다.

이는 image diffusion의 ε-prediction(예측 대상으로 noise를 쓰는 관행)과 정확히 같은 정신이다 — Treeffuser는 그 idiom을 tree learner에 맞게 가져온 것.

Experiments

합성 데이터에서의 표현력 + UCI 벤치마크 SOTA

5.1 합성 데이터: 어떤 분포든 잡아낸다

Multimodal

x에 따라 mode 개수가 변하는 응답. Treeffuser는 mode 수를 사전에 알지 못한 채 모든 x에서 정확히 mode들을 탐지.

Inflated

특정 점에 spike(zero-inflation 등)가 있고 support가 x에 따라 변하는 응답. peak와 가변 support 모두 회복.

Multivariate

출력이 다차원이고 covariance 구조가 복잡한 응답. Treeffuser가 dynamic correlation을 capture.

CRPS는 무엇인가? 왜 log-likelihood 대신 쓰는가?

CRPS(Continuous Ranked Probability Score)는 예측 CDF F와 관측값 y 사이의 거리이다:

CRPS(F, y) = ∫ (F(y') − 𝟙[y ≤ y'])² dy'

장점:
· proper scoring rule — true distribution에서 최소화됨
· sample만 있으면 추정 가능 — non-parametric 모델에 자연스러움
· log-likelihood처럼 tail에 민감하지 않다 — outlier 한두 개에 점수가 폭발하지 않음

논문에서는 추가로 MAE/RMSE도 보고하는데, Treeffuser는 점추정에서도 vanilla XGBoost/LightGBM과 거의 비슷하거나 동등 → 분포를 추정하면서 점추정 성능을 잃지 않는다.

결과 3: Newsvendor — 의사결정 가치로 환산

Inventory allocation 문제

매일 q개를 비용 c에 매입해 가격 p에 판매. demand y는 random. 기대 이익 최대화:

maxq p · 𝔼[min(q, y)] − c·q

최적해는 q* = F⁻¹((p−c)/p) — 즉 분포의 quantile. 50% 마진 가정 하에 각 모델이 학습한 p(y|x)로 quantile을 추정해 발주, 30일치 누적 이익 비교.

  • Treeffuser가 quantile regression을 큰 차이로 능가하고 NGBoost-Poisson과 비슷한 누적 이익을 달성.
  • "분포가 정확하면 의사결정도 정확해진다"는 점이 dollar 단위로 검증된 사례.
  • Treeffuser는 count data 전용으로 만들어진 게 아닌데도, count data 업무에서 dedicated 모델과 동급으로 작동한다.

Discussion

한계와 시사점, 그리고 후속 방향

왜 잘 되는가

Tree는 tabular data의 categorical/missing/mixed-scale을 자연스럽게 처리하고, robust하며 빠르다. Diffusion은 분포 가정 없이 임의의 conditional distribution을 학습할 수 있다. Tree의 inductive bias × Diffusion의 표현력이 합쳐지면서 두 약점을 동시에 메운다.

실용적 가치

CPU만으로 11.2만 샘플을 53초에 학습하는 가벼움 + 분포 가정 없음 + non-tuned default도 SOTA — 산업 현장의 tabular probabilistic prediction에서 곧바로 쓸 수 있는 baseline 후보. 코드 공개 (blei-lab/treeffuser).

한계 1 · 이산 응답

Diffusion은 이론적으로 continuous response를 가정. count data에 적용하면 동작은 하지만, 직접 modeling이 아니다. count-aware diffusion(예: discrete diffusion, jump SDE) 결합이 향후 과제.

한계 2 · 샘플링 비용

학습은 빠르지만 추론 시 SDE를 ~50 step 풀어야 함. 데이터당 많은 sample이 필요하면 비싸진다. progressive distillation, consistency model을 GBT에 적용하는 것이 후속 방향.

Take-away

"Diffusion은 GPU 위 neural network에서만 돌아간다"는 통념을 깬다. Score function을 GBT로 회귀해도 충분히 좋은 conditional distribution을 학습할 수 있고, tabular 영역에서는 오히려 NN 기반보다 빠르고 robust하다. 도구의 본질은 "score를 어떻게 추정할 것인가"이며, 그것은 데이터 modality에 따라 다른 학습기로 풀어야 한다는 관점의 전환이다.
